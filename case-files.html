Case Files

What happened before it became a lesson.

Case Files sit between Field Notes and Working Papers. They preserve observed conditions, findings, and related concepts without asking the evidence to become theory too early.

Case File 001

The Spreadsheet That Survived Migration

Status
Open reconstruction
Observed Condition
Legacy spreadsheet remained authoritative after platform migration.
Summary
The new system replaced storage but not trust. Operators continued validating outputs against the old file.
Finding
Migration succeeded technically while operational authority remained elsewhere.
Related Concepts
Spreadsheet Gravity, Stop Rediscovering Texas, Everything Is A Graph
Case File 002

The Helpful Person Problem

Status
Validated pattern
Observed Condition
One helpful operator became the routing layer for repeated exceptions.
Summary
The organization interpreted responsiveness as resilience. The response path was actually person-dependent.
Finding
Helpfulness can conceal missing process ownership.
Related Concepts
The Earl Problem, Vacation Risk, Operational Memory
Case File 003

The Dashboard Problem

Status
Active observation
Observed Condition
Metrics became more available than the operational reality they summarized.
Summary
Decision cycles optimized around visible indicators while unmeasured dependencies accumulated outside the frame.
Finding
Visibility is not equivalent to understanding.
Related Concepts
Executive Confidence, Conflicting Definitions, Everything Is A Graph
Case File 004

PowerPoint as Critical Infrastructure

Status
